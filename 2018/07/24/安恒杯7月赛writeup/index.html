第一道web

提示是order注入，测试了一下，发现是返回不同页面的盲注，直接丢sqlmap：

C:\Users\Administrator>sqlmap.py -u “http://101.71.29.5:10000/?order=name&button
=submit” -p order –thread 5

any)? [y/N]

第二道web

页面让输入密码，对了给flag。直接看源码：

最后一行给了提示 <!–hint.php–>

进hint看一下：

<?php
$key = “**”;
srand(time());

$a = rand(0,100);
$b = rand(0,100);
$c = rand(0,100);
$d = rand(0,100);
$e = rand(0,100);

$result = ((($a – $b)/$c)+$d) * $e;
$result = md5($key.$result.$key);
show_source(FILE);
?>

解读一下，这个是以时间戳为种子，产生5个随机数，然后运算，再和**组合，最后哈希一下就是答案，尝试在本地搭建phpstudy环境，输入这些代码，产生一个随机数：

fcfcb50465a5aa5bb47bae721213ecd0

将这个答案输进去：

页面显示

答案错误

再看源码：发现最后一行给了提示<!–50eb722bb4003cead5d6a5b58e7dc36e–>

把这个再输一下：还是错误，提示的哈希值又变了：<!–8503388b283a99a50fce1de30fe4ee43–>

想了想，应该是要用脚本提交哈希值后再提取他提示的值提交。

脚本：

flag{ef58f0b1e9437b4451248fb3f239622b}拿到了flag
