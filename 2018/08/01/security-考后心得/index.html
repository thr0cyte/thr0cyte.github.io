经过一个月的艰苦奋斗，终于将security+拿到手，这门考试说是注重技术实践，其实涉及面颇广，都是点到为止，而且并没有具体渗透实操的内容，但是对于了解安全整个行业的涉及面起到很大的作用。经过这次复习备考，我在巩固了渗透基础知识的同时，还掌握了企业中的安全产品实践、企业安全建设管理和密码学的知识，也算是“不虚此行”。下面将这次考试接触到的知识点总结在下，方便以后查阅和运用。


信息安全三大特性：confidentiality机密性、Integrity完整性、Availability实效性

虚拟机局域网VLAN：为了实现安全性，分割广播域和创建子网。通常运用在不同部门的网络划分。工作在数据链路层。

路由器：工作在网络层，连接因特网中各局域网、广域网的设备，可以根据acl过滤流量，分割数据包。可以识别网络层的逻辑地址信息，从而构建路由表，决策有效路径

传输介质：同轴电缆、双绞线（8根，4根备份，2根接收，2根发送）、光缆

端口号：

死亡之ping攻击：使用非法超大号的ICMP数据包

smurf攻击：发送一个icmp回应请求，源地址伪造成受害者的ip，目标地址为可ping通受害者的机器的ip，从而导致收到很多机器的icmp请求包，造成dos攻击。

arp偷毒：改变受害者系统的arp缓存表，欺骗它的流量经过攻击者主机再流向网关，从而进一步进行中间人攻击

dns协议，解析域名，TCP协议；SNMP协议，简单网络管理协议，对通信线路进行管理；SMTP协议，发邮件（配置邮箱服务器需关闭邮件转发）；POP和IMAP协议，收邮件；

ssh，sftp，scp都基于ssl

scp协议，安全文件拷贝协议，基于ssh协议，端口号22

VPN，虚拟专有线路，给两个在公网的机器建立传输加密通道。所需协议：四层：ssl；三层：IPsec；二层：PPTP,L2TP,MPLS

DMZ，非军事化区，用来隔离内网外网，放置各种用户访问的服务器，如web服务器，邮箱服务器

防火墙：一个信息安全的防护系统，允许或限制传输的数据通过。主要用于边界安全防护的权限控制和安全域的划分。现在最强的是状态防火墙，工作在网络、传输、应用层

UTM：同意威胁管理，集成了状态检测防火墙、VPN、网关防病毒、IPS、绿色上网、反垃圾邮件等安全防护功能，可以阻挡未授权的访问，网络入侵，病毒，蠕虫，木马，间谍软件，钓鱼诈骗，垃圾邮件等安全威胁。

URL Filters过滤网关，又称内容过滤器，控制公司职员访问网络资源，防止威胁

waf：web层的防火墙，狗、云、盾等，防护对web应用的攻击

IDS，入侵检测系统（被动）；IPS入侵防护系统（主动）

蜜罐：吸引攻击者攻击，有利于安全人员分析攻击手段和诱捕攻击者

WEP，使用RC4加密，容易被破解

WPA，tkip模式，支持eap，容易被破解

WPA2，CCMP或CCM模式，AES加密，目前最安全的无线加密

802.1X：内网接入身份验证协议

胖ap：家用小型wifi；瘦ap：企业、校园用等大型wifi，可进行认证登陆

战争驾驶：wifi侦察；战争标记：标记弱安全的wifi；双面恶魔：建立假ap，转发请求；蓝牙劫持：用于发送广告；蓝牙攻击：用于窃取手机机密信息

PII，个人可识别信息，手机号、生日、家庭住址等信息

信息安全管理要求：欧盟：GDPR 美国：HIPA 中国：ISO27001

用户密码策略：最小长度、密码历史、最长时间、最短时间、密码复杂度（大小写字母+数字+特殊符号）等

MOU：谅解备忘录，双方经过协商、谈判达成共识后，用文本的方式记录下来，简称协议

SLA，服务水平协议，由服务供应商与用户签署的法律文件，承诺提供的服务支持

人力资源相关策略：职责分离：不应有人从头到尾完全控制一项敏感任务；最小特权；工作轮换：不允许某人过长时间担任某个职务；强制休假

安全意识培训：密码习惯、数据处理、清理桌面、尾随防范、个人设备、使用社交网络

ALE，年度损失期望，等于单次损失金额×年化损失概率

风险处置策略：降低风险、规避风险、转移风险（保险）、接受风险、威慑风险

物理安全控制：锁、CCTV监控、保安、双重门、栅栏立柱、报警器、门禁等

环境控制：温度太高，服务器会重启或宕机；温度太低，机器结霜；湿度太高，机器会锈；湿度太低，容易静电。

业务连续性计划：BIA（核心内容），业务影响分析； MTD，最高可承受的停机时间； RPO，恢复多久前的数据； RTO，还原时间目标

备用站点：热站：无实时数据，出事送硬盘就可以；温站：几天才能恢复，基础设备，没服务器；冷站：什么都没有的一个空间； 移动数据中心，很昂贵

两地三中心：两地是指同城、异地，三中心是指生产中心、同城容灾中心、异地容灾中心。

接班人计划：确保关键业务人员有接班人

负载均衡：大家的活一起干，分摊压力。集群：自己的活忙不过来给他人分担，实现高可用性

灾难恢复流程：确认真实性和影响面，通知领导，启用灾难恢复，遏止事态发展，恢复，报告，总结

备份：完全备份：备份所有文件； 差异备份：备份上次完全备份后改变的文档； 增量备份：备份上次完全备份和差异备份后改变的文档

易失性顺序：CPU缓存、RAM、硬盘内存、硬盘驱动

DLP，数据丢失防护，检测并防止数据被发送到未经授权的机关

社会工程学类型：欺骗、扮演、恐吓、钓鱼（鱼叉、吊鲸）、肩窥、垃圾搜寻、尾随、垃圾邮件

攻击手段：DDOS，软件攻击（缓冲区溢出），web攻击（xss、sql注入、csrf、命令注入、目录包含等）

恶意软件：广告软件、间谍软件、逻辑炸弹、木马、蠕虫、rookits、后门、勒索病毒等

TPM，可信平台模块，计算机加密芯片，存储加密密钥

可接受使用策略：进公司签订，对数据进行保护

Fuzzing，模糊测试，是一种通过向目标系统提供非预期的输入并监视异常结果来发现软件漏洞的方法

移动设备安全控制：设备管理、设备加密、远程擦除、锁定、屏幕锁、GPS定位、应用控制

多因素身份认证：sth you have ; sth you know ; sth you are ;sth you do

HOTP，一次性密码，u盾之类的东西。 TOTP，时钟同步令牌

假阳性：误报 ； 假阴性：漏报

单点登陆SSO,输入一次凭证，能够访问主次网络域中的所有资源

认证协议：

PAP 是点对点认证协议，明文传输
CHAP 是点对点认证协议，其周期性的通过3 次握手协议验证客户端身份

Kerberos 是C/S 模式协议，同时使用TCP/UDP 协议，提供了服务器端与客户端的相互认证，依靠’tickets’允许节点在不安全网络上验证身份并工作。其默认使用端口88。Kerberos 密钥分发中心（KDC）通过来使用安全的加密密钥和tickets 来确认身份和授权接入网络。Kerberos 被微软选为Windows 2000 和随后Active Directory domains的默认认证协议。

RADIUS 用于远程接入网络，运行在应用层，使用UDP 协议。大多数控制访问网络的网关都有与RADIUS 服务器进行通信的RADIUS 客户机组件。RADIUS server 是保存用户账户和密码认证数据库。RADIUS 使用共享密钥（对称密钥）和经过MD5 HASH 的传输密钥。目前主要使用PAP/CAHP 加密认证。可以提供使用网络服务的用户提供远程集中身份验证、授权和计费管理。

Diameter 系替代RADIUS 的AAA 协议，使用TCP 协议，通过tcp 或sctp 提供可靠的传输，通过ipsec 和tls 来保证传输的安全性。其为各种应用协议（包括扩展认证应用协议EAP）提供了一个基本框架，它定义了协议的传输机制、消息格式、消息处理、差错处理、计费与安全服务等。

TACACS+默认使用TCP 端口49，并且不兼容TACACS 和XTACACS。允许客户端接受用户名和密码，并发送一个查询给TACACS 认证服务器。由认证服务器决定是否接受或拒绝身份验证请求并发送响应，客户端依据此接受或拒绝验证请求。

LDAP，轻量级目录访问协议，是X.500 标准的一个简单子集，也被称为X.500-Lite，使用TCP 协议，用于无线客户端和RADIUS server 之间，特性包括动态WEP 密钥（或TKIP）和相互认证。LDAP 服务器称为目录系统代理（DSA），默认端口为TCP 和UDP 端口389，LDAPS（SSL）默认为端口636，全球目录端口默认为3268，LDAPS 为3269。LDAP可以用SSL/TLS 进行加密，即LDAPS。
XML 系可扩展标记语言，是一种用于标记电子文件使其具有结构性的标记语言。

SAML 系安全断言标记语言，基于XML 的一个开放标准的框架协议，用于在不同的安全域(security domain)之间交换认证和授权数据。

访问控制模型：自主访问控制、强制访问控制、基于角色的访问控制

加密算法：

Blowfish，系64bits 分组及可变密钥长度（from 32 bits to 448 bits）的对称密钥分组密码算法，密钥最小，速度最快（超越DES）。

Twofish，替换DES 算法的高级加密标准（AES）算法的候选算法, 使用分组加密机制, 128bits 数据分组，128/192/256 bits 可变长度密钥。

ECC，椭圆加密算法，密钥长度较短且不定，随加密强度的提高，密钥长度变化不大。ECC 160bits 密钥加密强度和RSA 1024bits 密钥加密强度相近。

ECDHE：Elliptic curve Diffie–Hellman,匿名密钥协商协议，允许双方各有一个椭圆曲线公钥-私钥对，以在不安全的信道上建立一个共享的秘密。同时提供了CRC 完整性检查和RCA 加密。

Diffie–Hellman，密钥交换协议，只能用于密钥的交换，而不能进行消息的加密和解密。

DES，系64bits 分组及56bits 密钥的对称密钥加密算法；虽然已被3DES 和AES 取代，但是目前依然在使用中。
3DES，系64bits 分组及168bits 密钥的对称密钥加密算法;其没有ECC 安全,但是计算速度更快。

AES，高级加密标准，也称Rijndael 加密算法（区别如下），用来替代DES，AES 分组长度为128bits，密钥长度可以是128，192 或256bits。比RC4 加密强度更高，加密速度更快。已成为对称密钥加密中最流行的算法之一。

RSA：公钥加密算法，能同时用于加密和数字签名。RSA 密钥一般推荐使用1024bits，递增为1024 的整数倍。

RC4：流加密算法，密钥大小范围为40 至2048bits，默认为256 字节，算法速度可达到DES 加密的10 倍左右。常用于WEP/WPA 加密。

HTTPS：安全套接字层超文本传输协议，即在HTTP 下加入SSL/TLS 层以强化安全。

S/MIME 是用于电子邮件和其他电子消息通信的应用程序。其为电子消息应用程序提供以下加密安全服务：身份验证、消息完整性、不可抵赖性（使用数字签名）、隐私和数据安全（使用加密）。

PGP(Pretty Good Privacy)，是一个基于RSA 公钥加密体系的邮件加密软件。提供保密性和完整性

MD2/4/5：HASH 函数，创建128bits 的HASH 值。

SHA:安全HASH 算法，不同的版本产生不同长度的消息摘要，SHA-1 产生160bits 的HASH 值，SHA-1，SHA-224 和SHA-256 适用于长度不超过2^64 二进制位的消息。SHA-384 和SHA-512 适用于长度不超过2^128 二进制位的消息。

HMAC，密钥相关的哈希运算消息认证码，以一个密钥和一个消息为输入，生成一个消息摘要作为输出。使用哈希散列算法与对称密钥，HASH 提供数据完整性、对称密钥提供真实性。可以用于身份认证，常被用于IPSec 协议（其他包括：BlowFish/TwoFish）。

PKI 是一套硬件、软件、人员、政策和程序,用于创建、管理、分发、使用、存储和撤消数字证书，使用双向信任模型与CAs 建立信任关系。

CSR，证书签名请求文件，创建CSR 时，需要先生成自己的密钥对(公钥和私钥)，保存基于RSA 的私钥，并把公钥及部分个人身份信息传送给CA，CA 验证后将发给用户一个数字证书，该证书内包含用户的个人信息和他的公钥信息，同时还附有认证中心的签名信息。删除CSR 即删除了自己的私钥，在未获取证书前不得删除CSR。

数字证书，一个经证书授权中心数字签名的包含公开密钥拥有者信息以及公开密钥的文件，以数字证书为核心的加密技术(加密传输、数字签名、数字信封等安全技术)可以对网络上传输的信息进行加密和解密、数字签名和签名验证，确保网上传递信息的机密性、完整性及交易的不可抵赖性。

数字签名，是非对称密钥加密技术与数字摘要技术的应用，用于鉴别数字信息的完整性，同时具有不可否认性。

OCSP：在线证书状态协议，用于获得X.509 证书的吊销状态。一个OCSP 响应器可以返回响应表示请求中指定的证书状态是’好’，’撤销’，或’未知’的。
