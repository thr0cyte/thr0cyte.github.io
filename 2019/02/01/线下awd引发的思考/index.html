前言

前一段时间，参加了公司举办的线下awd比赛。第一次接触这样的赛事，研究了一番发现是红蓝对抗，顿时兴致大起。比赛后，收获颇多，特有此文。

比赛内容为10支队伍，每支队伍的靶机都是一个相同的cms系统，进攻别人靶机拿flag的同时，还要修复自己的漏洞。

就先谈谈防守方，众所周知，在网络攻防中防守方永远都是很被动的一方，然而这次比赛让我发现，防守其实也可化为主动。
首先要思考一个问题，防守到底防的是什么。第一，防守拥有系统存在的漏洞并修复，所以防守即是进攻。第二，清除攻击者留下的后门，不知攻焉知防？第三，增添防护设备或软件措施。换句话说，第一点即渗透测试，第二点即应急响应，第三点即安全开发。

围绕这三点，谈一谈这次比赛中用到的技巧。

流量抓取分析漏洞点：

采用tcpdump -i eth0 tcp port 80 -w a.cap将所有访问流量抓取下来并放入wireshark中分析。
可以看到，攻击者登陆了cms，并上传了一个后缀名为pht的后门文件。通过分析，发现了admin.php的文件上传漏洞，从而在代码中修复黑名单，增加pht后缀。
这个http会话流量可以看到，down.php存在任意文件下载漏洞，在filename参数中加入过滤代码即可修复

监控文件

代码层waf

如何使用：

有root权限
vim php.ini
auto_append_file = “/dir/path/phpwaf.php”
重启Apache或者php-fpm就能生效了。
也可以写在 .user.ini 或者 .htaccess 中。
php_value auto_prepend_file “/dir/path/phpwaf.php”

只有user权限
没写系统权限就只能在代码上面下手了，也就是文件包含。
这钟情况又可以用不同的方式包含。
如果是框架型应用，那麽就可以添加在入口文件，例如index.php，
如果不是框架应用，那麽可以在公共配置文件config.php等相关文件中包含。
include(‘phpwaf.php’);
还有一种是替换index.php，也就是讲index.php改名为index2.php，然后讲phpwaf.php改成index.php。
当然还没完，还要在原phpwaf.php中包含原来的index.php。
index.php -> index2.php

phpwaf.php -> index.php

include(‘index2.php’);

修改curl

alias curl=’echo fuckoff’ 权限要求较低
chmod -x curl 权限要求较高
防止对方curl获得flag，用不到实战中，一笔带过

不死马

防守分享完，记录下我最喜欢的进攻环节
权限维持是后渗透测试必要的环节，不死马起到关键作用：

访问此脚本即可启动不死马，市面上的不死马基本上都是在内存中循环写入一句话实现这种形式。

有攻便有防。重启apache；创建和不死马相同名称的文件夹；ps aux | grep www-data | awk ‘{print $2}’ | xargs kill -9都可以破解此不死马
